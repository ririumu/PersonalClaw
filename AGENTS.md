# PersonalClaw Instructions

PersonalClaw is a personal, local, full-access Codex environment. It is not an OpenClaw-style platform that should be expanded by default.

## Working Assumption

- Treat this repository as intentionally minimal and potentially already complete.
- Default to refining the framing, documentation, and operating assumptions before proposing new implementation.
- Do not add scaffolding, frameworks, placeholder code, or speculative architecture unless explicitly requested.
- Prefer small, durable artifacts such as README updates, instructions, notes, and narrowly useful scripts.

## Repository Intent

- `README.md` explains the concept to humans.
- `AGENTS.md` carries the standing repository instructions for Codex.
- Additional files should earn their place by improving real usage, not by making the repository look fuller.

## Editing Style

- Keep changes direct, compact, and concrete.
- Preserve the idea that a low file count can be the correct finished state.
- If a request implies turning PersonalClaw into a larger framework, challenge that assumption before building.
